# 序列建模

序列建模就是对有顺序关系的数据进行建模，利用前面的数据预测后面的数据。
不同的序列模型都在做一件事，给序列每个元素打分或分配权重，然后利用这些权重进行加权求和，来建立样本与样本之间的关联。

# RNN
循环神经网络能捕捉时间序列中的依赖关系，其核心特点是通过循环连接使得每个时刻的输出不仅依赖于当前输出，还依赖于先前的状态。这样RNN就能处理更长的序列数据。

RNN的优缺点
优点：
能够处理变长序列
有效地捕捉前后数据点之间的依赖关系，适合于需要考虑上下文信息的任务。
应用广泛：RNN被广泛应用于自然语言处理、语音识别、时间序列预测等任务中。
缺点：
梯度消失与爆炸问题：在训练过程中，由于梯度需要通过多层的链式法则进行反向传播，RNN容易遇到梯度消失或梯度爆炸的问题，导致难以捕捉长序列的依赖关系。
训练效率低：RNN的计算过程是逐步进行的，这使得其训练速度相对较慢，且在长序列时训练更加困难。

# LSTM
LSTM在标准RNN的基础上引入了门控机制，通过记忆单元来控制信息的流动，使得LSTM能够捕捉更长时间范围的依赖关系。LSTM可以选择性记忆，记住重要的时期，对不重要的事情选择性遗忘

输入门：决定输入的信息是否更新到记忆单元中，输入门包含一个Sigmoid激活函数，决定哪些信息要更新，还包含一个tanh激活函数，生成新的候选值。
遗忘门：决定遗忘多少之前的记忆，遗忘门的作用是过滤掉不需要的信息。遗忘门的输入是当前的输入和上一时间的隐藏状态，
输出门：决定记忆单元中有多少信息需要输出，通过一个Sigmoid函数生成输出门的控制值


LSTM的优缺点
优点：
解决梯度消失问题：LSTM通过引入门控机制，允许模型保留长期信息，能够有效地捕捉长时间依赖关系。
长序列建模：模型可以有效地记住重要的信息，并能够忽略不相关的信息（遗忘门）
灵活性：LSTM能够处理各种类型的序列任务，如自然语言处理（NLP）、语音识别、时间序列预测等。
缺点：
计算复杂度较高
难以捕捉超长时间的依赖：处理非常长的序列时，可能面临记忆溢出的问题

```python
nn.LSTM(
    input_size,     # 输入的向量维度
    hidden_size,    # 隐藏状态的维度 
    num_layers=1,   # 几层
    bias=True,
    batch_first=False,
    dropout=0,  # 随机丢弃多少样本
    bidirectional=False
)
```
LSTM是如何解决梯度消失的问题的？
LSTM实际上并没有完全解决梯度消失问题，而是通过记忆单元和门控机制，使梯度在时间维度上的传播路径减少了反复的非线性变换和矩阵乘法，从而显著缓解梯度消失。
在单元状态的传播路径上，梯度主要通过逐元素乘法（与遗忘门相关）进行传递，因此相比普通RNN中连续的权重矩阵相乘，不会轻易发生指数级衰减。


# GRU
GRU通过两个主要的门控机制来控制信息流：重置门和更新门。这两个门决定了哪些信息应该被忘记，哪些信息应该被保留或更新。

重置门控制着如何使用过去的隐藏状态来计算当前时刻的候选状态。如果重置门的值接近0，则表示忽略过去的隐藏状态，仅依赖当前输入。
更新门决定了当前的隐藏状态有多少来自于上一时刻的隐藏状态，多少来自于当前的候选状态。

优点：
较少的参数：GRU 比 LSTM 更简单，参数较少，计算效率更高。
较快的训练速度：GRU 的门结构较简单，因此训练时间比 LSTM 更短。
长距离依赖建模：GRU 能够有效地捕捉时间序列数据中的长距离依赖，解决传统 RNN 的梯度消失问题。

缺点：
性能不如 LSTM：在一些复杂任务中，LSTM 可能比 GRU 更强大，尤其是在处理更复杂的时间序列任务时。
较少的控制：GRU 只有两个门（与 LSTM 的三个门相比），可能在某些情况下缺乏足够的灵活性。


RNN，LSTM，GRU的共同点和不同点：
相同：
都能够处理序列数据，都有隐藏状态，都能处理历史信息
不同：
RNN没有门控机制，结构比较简单，容易出现梯度消失和爆炸
LSTM增加了记忆单元和门控机制，能够保存长期信息，能够筛选信息，梯度更稳定
GRU相当于简化版的LSTM，参数更少，结构相对简单，训练更快
