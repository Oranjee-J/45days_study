# Titanic Survival Prediction Project

## Dataset Description

数据集包含891条乘客记录。
主要特征包含：
'survived'：是否存活
'pclass'：船舱等级
'age'：年龄
'sibsp'：兄弟姐妹/配偶数量
'parch'：父母/子女数量
'fare'：船票价格
'embarked'：登船港口

## EDA

数据概况：
Titanic 数据集共包含 891 条样本和 15 个特征

缺失值分析：
deck缺失688条
age缺失177条
embarked缺失2条

主要发现：
- 女性存活概率高于男性
现实依据：泰坦尼克救援过程中遵循‘妇女儿童优先’的原则，因此女性乘客生存率更高
- 高等级船舱乘客存活概率更高
现实依据：头等舱乘客的经济地位和社会地位更高，救援优先级更高，因此生存率也更高
- 票价较高的乘客生存率
现实依据：同上
- 存活者平均年龄更年轻
现实依据：一是儿童被优先救援，二是泰坦尼克失事海域海水冰冷，年轻人身体强壮，生存率更高。

## 模型训练
1. 训练集与测试集的划分
将处理好的数据集按照8：2的比例划分为训练集和测试集，随机种子为42，确保结果可复现。
为了保证训练集和测试集中类别分布一致，采用stratify=y进行分层抽样。

2. 模型选择
本实验选择以下三种模型进行比较：
- 逻辑回归模型 Logistic Regression
逻辑回归是一种经典的线性分类模型，广泛用于二分类任务。训练速度快，模型结构简单，可解释性强。
- 随机森林分类器 Random Forest Classifier
随机森林分类器是一种基于决策树集成的Bagging方法，能够解释非线性的复杂关系，鲁棒性较好，不容易过拟合。
- 梯度提升树 Gradient Boosting Classifier
梯度提升树是一种Boosting集成方法，能够通过不断训练新的决策树来修正前一轮的预测误差，梯度提升树能学习特征之间的交互影响，提高预测的精度，在中小型数据集上表现良好；Titanic数据集里的因素之间存在复杂的交互关系，梯度提升树能很好地学习。

3. 超参数调优
从sklearn库导入了GridSearchCV方法进行网格搜索，并使用了5折交叉验证进行超参数调优。
参数搜索范围如下：
```python
{
    "n_estimators":[50,100,200],
    "max_depth":[3,5,7,None],
    "min_samples_split":[2,5,10]
}
```
最优参数结果：
```python
{
    'max_depth': None,
    'min_samples_split': 10,
    'n_estimators': 100
}
```
最优交叉验证得分：0.7554
结果说明，适当提高节点分裂所需样本数能够有效降低模型过拟合风险。

三个模型的准确率都很接近，逻辑回归和梯度提升的准确率为最高，是80.44%。
梯度提升模型的精确率最高，为80.35%，这表明梯度提升树预测生还的人中，确认生还的比例最高。
随机森林的召回率最高，是71.01%。这表明随机森林预测出生还的人数最多。
随机森林的F1分数也最高，是73.13%，这表明随机森林模型各方面指标的平衡性最好。
因为随机森林模型各方面综合表现最好，因此选择随机森林模型作为最终超参数调优的模型。

## 错误分析
|实际情况|预测死亡|预测生存|
|-------|-------|-------|
|实际死亡|94|16|
|实际生存|20|49|

通过分析预测错误的样本发现：
- 部分三等舱乘客最终幸存，但模型预测其死亡；
- 部分一等舱乘客最终死亡，但模型预测其生存；

错误原因分析：
虽然一等舱乘客存活的概率比三等舱乘客存活的概率更高，但是船舱等级本身对乘客存活概率影响的程度就不大，而且一等舱乘客存活的概率也不超过50%（较低），因此有部分乘客虽然身处一等舱但没有存活，却被误判成存活，而部分乘客身处三等舱存活，却被误判成死亡。
